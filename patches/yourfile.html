evoke_harmony_convolution

evoke_harmony_convolution – evoke some harmonic content from an input signal.

Description

Evokes harmonic content from an input signal by convolving it with an impulse response containing the harmony in the form of a grain of sound containing each octave of one or more pitch-classes.

Syntax

a_output evoke_harmony_convolution k_kernel_duration, i_impulse_gain, 
  i_dirac_level, i_pitch_class_1, i_pitch_class_2, i_pitch_class_3, 
  i_pitch_class_4, i_pitch_class_5

Initialization

i_impulse_gain – the impulse response is multiplied by this before use.

i_dirac_level – the amplitude of the first sample of the grain. If not 0, transmits the original input signal to the output signal at the specified gain.

i_pitch_class_1 through i_pitch_class_5– a sinusoid is added to the impulse response at each octave of the pitch-class, from MIDI key numbers 0 through 127. Note that the pitch-classes are not limited to integers in [0, 12), but can be any pitch in that range. The impulse response is normalized to the L2 norm of amplitude, and the impulse gain and Dirac level are relative to that. The pitch-classes are optional; parameters that are not specified or not greater than -1 are not used.

Performance

a_output – the monophonic audio output signal that results from convolving a_input with the impulse response containing the harmony.

k_kernal_duration – the duration of the impulse response grain; determines the amount of time that energy in the input signal will ‘ring’ from the harmony in the impulse response.

a_input – a monophonic audio input signal.

Note that harmonic content can be evoked from the input signal only when there is at least some energy at some octave of one of the pitch-classes.
